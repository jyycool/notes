## 【DP】 第1章 设计模式简介

面向对象的设计原则也被称为 ***SOLID***. 在设计和开发软件的时可以应用这些原则, 以便创建易于维护和开发的程序. ***SOLID*** 最初是由 Robert C.Martin 所提出的, 它们是敏捷软件开过程中的一部分. 

***SOLID*** 原则包括了:

1. 单一职责原则
2. 开闭原则
3. 里式替换原则
4. 接口隔离原则
5. 依赖倒置原则

### 一、单一职责原则

该原则指出软件设计模块应该只有一个被修改的理由. ***在大多数情况下, 编写 Java 代码时都应该将单一职责原则应用于类.*** 



### 二、开闭原则

模块、类和函数应该对外扩展开放, 对修改关闭
